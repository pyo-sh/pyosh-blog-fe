# Project context from docs

When a task depends on previous work, technical decisions, or known issues, read the relevant docs first.

Available indexes:

- `docs/{area}/progress.index.md`
- `docs/{area}/findings.index.md`
- `docs/{area}/decisions.index.md`

Area notes:

- `workspace` covers root repo changes such as tools, Docker, tmux, skills, and workflow config
- CI or CD changes belong under the relevant app area, not `workspace`
